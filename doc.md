
# Инструкция по развертыванию и управлению данными

## 1. Настройка окружения разработки

### Запуск инфраструктуры
Поднимите контейнеры с базой данных и приложением:

```bash
docker-compose up -d --build
```

Проверьте статус сервисов. `football-postgres` должен иметь статус **Healthy**, а `football-app` — **Up**:

```bash
docker-compose ps
```

### Синхронизация локальной среды
Установите зависимости на хост-машине и сгенерируйте типы Prisma для корректной работы автодополнения в IDE:

```bash
npm install
npx prisma generate
```

> ** Важно:** После генерации типов перезапустите сервер TypeScript в VS Code, чтобы исчезли ошибки подсветки синтаксиса:
> Нажмите `Ctrl+Shift+P` (или `Cmd+Shift+P`) → выберите **"TypeScript: Restart TS Server"**.

### Применение миграций
Примените накопившиеся изменения схемы к базе данных PostgreSQL внутри контейнера:

```bash
docker-compose exec app npx prisma migrate deploy
```

## 2. Создание тестовых данных

Выполняйте команды строго в указанном порядке:

1.  Убедитесь, что контейнеры запущены:
    ```bash
    docker-compose up -d --build
    ```
2.  Примените актуальные миграции:
    ```bash
    docker-compose exec app npx prisma migrate deploy
    ```
3.  Запустите скрипт заполнения базы (используется `tsx` для поддержки ESM):
    ```bash
    docker-compose exec app npx tsx scripts/seed.ts
    ```
4.  Обновите локальные типы Prisma:
    ```bash
    npm install && npx prisma generate
    ```
    *Не забудьте снова выполнить "TypeScript: Restart TS Server" в VS Code.*

## 3. Очистка базы данных

> **ВНИМАНИЕ:** Данная операция удаляет все данные безвозвратно. **Никогда не запускайте этот скрипт на продакшене!** Используйте только в локальной разработке или на тестовых стендах.

Очистка всех таблиц с сохранением структуры БД:

```bash
docker-compose exec app npx tsx scripts/reset-db.ts
```

После очистки рекомендуется сразу наполнить базу заново:

```bash
docker-compose exec app npx tsx scripts/seed.ts
```

Проверка текущего IP чтоб зайти с браузера телефона
```bash
ip addr show | grep "inet " | grep -v 127.0.0.1 | awk '{print $2}' | cut -d/ -f1 | head -1
```
